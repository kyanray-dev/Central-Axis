# 中轴寻迹 (Central-Axis Trace) - AI 小程序原型

这是一个关于“北京中轴线”的交互式 AI 小程序原型演示。通过 AI 吉祥物“小衡”的引导，为游客提供个性化的中轴线游览体验。

## 📱 核心交互逻辑流程

本原型采用单页应用（SPA）式的翻页交互，模拟移动端小程序的体验逻辑。核心流程分为以下几个关键模块：

### 1. 首页：走近中轴线 (Home)
- **展示内容**：中轴寻迹品牌视觉与 AI 吉祥物“小衡”登场。
- **交互逻辑**：
  - 点击“开始寻迹”按钮，开启个性化定制流程。
  - 页面会记录用户的访问状态，若已开始则显示“继续寻迹”。

### 2. 偏好选择 (Preference Selection)
- **展示内容**：三步生成个性化游览方式的问卷。
- **交互逻辑**：
  - **怎么玩**：轻松逛 / 深度看 / 任务挑战
  - **感兴趣**：古建筑 / 历史故事 / 声音体验 / 拍照打卡
  - **讲解方式**：简短版 / 深度版 / 故事版
  - 用户点击标签进行单选，系统（小衡）会实时记住这些偏好设定，点击“生成路线推荐”进入下一步。

### 3. 路线推荐 (Route Recommendation)
- **展示内容**：根据用户偏好生成的 4 条特色中轴线路线：
  - **听见时间**：钟鼓楼、万宁桥、什刹海
  - **看见礼序**：景山、故宫、太庙、社稷坛
  - **走过中轴**：天安门、天安门广场、正阳门
  - **回到天地**：天坛、先农坛、永定门
- **交互逻辑**：用户点击任意一条路线卡片，系统会进入“模拟定位与 AI 讲解”模式。

### 4. AI 实时讲解 (AI Guide)
- **展示内容**：基于当前选择路线的模拟 GPS 定位与景点讲解。
- **交互逻辑**：
  - 模拟获取当前位置，显示附近的景点（如：太和殿、中和殿等）。
  - 根据用户在“偏好选择”中设定的“讲解方式”（简短/深度/故事），展示对应的讲解文案。
  - 提供快捷提问按钮（如“这个纹样有什么含义？”），模拟与 AI 助手的实时对话互动。

### 5. 限时活动：古建纹样寻宝 (Activity/AR)
- **展示内容**：互动收集类游戏，鼓励用户观察古建筑细节。
- **交互逻辑**：
  - 包含三种核心纹样卡片：藻井、斗拱、屋脊兽。
  - **拍照识别**：调用摄像头（或模拟界面），小衡 AI 实时“观察”并识别纹样。
  - **收集机制**：识别成功后解锁对应的卡片，全部集齐可获得专属徽章。

### 6. 个人中心 (Mine)
- **展示内容**：用户的游览数据与成就展示。
- **交互逻辑**：
  - 统计已走路线数、已收集纹样进度、已获得徽章。
  - **分享功能**：点击生成包含用户路线、卡片和徽章的中国风专属“中轴寻迹卡”。

## 🛠️ 技术实现
- **HTML5 + CSS3**：采用原生 Web 技术构建，利用 Flexbox 和 CSS Variables 实现响应式与主题样式。
- **Vanilla JavaScript**：纯原生 JS 处理页面切换、状态管理（偏好记忆）、模拟 AI 回复以及摄像头调用逻辑。
